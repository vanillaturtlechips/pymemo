## ✅ 논리적 사고력 극대화 학습 체크리스트

이 체크리스트는 문제를 분석하고 코드를 작성하는 **3단계**를 순서대로 정리한 것입니다.

### 1단계: 📝 문제 분석 및 데이터 정의 (최소 5분 투자\!)

이 단계는 **코드를 한 줄도 치지 않고** 종이나 메모장에서 수행합니다.

  * **[ ] 핵심 목표 파악:** 이 문제가 궁극적으로 '무엇'을 요구하는가? (예: 개수 세기, 합치기, 빠진 원소 찾기 등)
  * **[ ] 입력 데이터 구조 분석:**
      * **N, M, X** 등의 변수는 \*\*정수(int)\*\*인가?
      * 수열 $\mathbf{A}$는 **1차원 리스트**인가? **2차원 행렬**인가?
      * 입력이 \*\*줄 바꿈(`\n`)\*\*으로 구분되는가, \*\*공백(`     `)\*\*으로 구분되는가?
  * **[ ] 출력 형식 확인:**
      * 원소들이 **공백**으로 구분되어야 하는가?
      * 결과가 **여러 줄**에 걸쳐 출력되어야 하는가?
  * **[ ] 적합한 자료 구조 선택:** 이 문제의 특성상 **List**가 유리한가, **Set**(중복 제거/차집합), **Dictionary**(검색/매핑)가 유리한가?

\<hr\>

### 2단계: 🗺️ 의사 코드(Pseudocode) 설계 (가장 많은 시간을 투자\!)

가장 논리적인 순서를 **사람의 언어**로 설계하는 단계입니다.

  * **[ ] 초기화:** 필요한 변수와 리스트(예: `count = 0`, `result = []`)를 정의했다.
  * **[ ] 반복 구조 설계:** 몇 번의 `for` 문이 필요한가? (예: 행렬 덧셈처럼 $\mathbf{N} \times \mathbf{M}$ 이중 반복이 필요한가?)
  * **[ ] 핵심 로직 위치:** **`if` 조건문**은 **반복문 안**에 있어야 하는가, **밖**에 있어야 하는가?
  * **[ ] 최종 단계 정의:** 최종 계산된 값을 **어떻게 출력**할지(정렬, `print(*list)` 등) 순서대로 명확히 했다.

### 3단계: ⌨️ 코드로 변환 및 디버깅

설계한 논리를 파이썬 문법으로 옮긴 후 점검합니다.

  * **[ ] 문법 정확성 확인:** `input()`과 `int()` 사용, 괄호 `()`의 개수, 변수 이름의 오타(`intput` 방지) 등을 점검했다.
  * **[ ] 경계 조건 테스트:** $\mathbf{N}$과 $\mathbf{M}$이 가장 작을 때(예: $1$) 코드에 오류가 없는지 머릿속으로 확인했다.
  * **[ ] 디버깅 및 분석:** \*\*에러(Error)\*\*가 발생하거나 \*\*틀린 답(Wrong Answer)\*\*이 나왔을 때, 코드를 수정하기 전에 **2단계 의사 코드와 비교**하며 논리적 오류 지점을 먼저 찾아냈다.
  * **[ ] 풀이 정리:** 문제가 해결된 후, 더 \*\*효율적인 Python 내장 함수(예: `.count()`, 리스트 컴프리헨션)\*\*를 사용하여 풀 수 있는지 확인하고 정리했다.
